# Сервер на домашнем ПК (Windows 11, белый IP)

Внешний адрес: **176.99.158.181** (сигналинг WebSocket — **ws**, не https в URL для порта 9090).

## Что должно крутиться на ПК

| Сервис           | Порт  | Назначение                          |
|------------------|-------|-------------------------------------|
| `signaling-server` | **9090** | WebRTC сигналинг (обязательно)    |
| `backend` (опц.) | **3001** | API с токеном (если используете)   |

## 1. Установить Node.js LTS

1. Скачать с https://nodejs.org/ (LTS).
2. Установить, перезапустить PowerShell.
3. Проверить:
   ```powershell
   node -v
   npm -v
   ```

## 2. Скопировать проект на Windows ПК

Скопировать папки на ПК (USB, сеть, git clone):

- `signaling-server\` — минимум для связи ведущий↔ведомый
- при необходимости `backend\`

Структура на ПК, например:
```
C:\RomanAlexandrov\signaling-server\
C:\RomanAlexandrov\backend\
```

## 3. Запуск сигналинга

В **PowerShell от имени администратора** не обязательно; обычный пользователь подойдёт.

```powershell
cd C:\RomanAlexandrov\signaling-server
npm install
npm start
```

Должно появиться:
```
Сервер сигналинга слушает порт 9090
Подключение: ws://<IP_ЭТОГО_КОМПЬЮТЕРА>:9090
```

Оставить окно открытым или настроить автозапуск (см. ниже).

## 4. Открыть порт в брандмауэре Windows

**Правило для входящих подключений на порт 9090:**

1. Панель управления → Брандмауэр Защитника Windows → Дополнительные параметры.
2. Правила для входящих подключений → Создать правило.
3. Тип: **для порта** → TCP → **9090**.
4. Действие: **разрешить подключение**.
5. Профиль: все (Частная/Общая — по желанию).
6. Имя: `RomanAlexandrov Signaling 9090`.

Если используете backend на 3001 — аналогичное правило для **3001**.

Или в PowerShell **от администратора**:

```powershell
New-NetFirewallRule -DisplayName "Signaling 9090" -Direction Inbound -Protocol TCP -LocalPort 9090 -Action Allow
```

## 5. Порт на роутере (если есть NAT)

Если у ПК не прямой белый IP, а белый IP у роутера — на роутере включить **проброс порта (port forwarding)**:

- Внешний порт **9090** → IP вашего ПК в LAN, порт **9090**, протокол TCP.

Проверка с другого устройства (телефон в мобильном интернете):

```bash
# с Mac/Linux — проверка что порт открыт (если установлен nc)
nc -vz 176.99.158.181 9090
```

## 6. Автозапуск при включении ПК (по желанию)

**Вариант A — Планировщик заданий**

1. `Win + R` → `taskschd.msc`
2. Создать задачу → Триггер: «При входе в систему».
3. Действие: запуск программы:
   - Программа: `C:\Program Files\nodejs\node.exe`
   - Аргументы: `C:\RomanAlexandrov\signaling-server\server.js`
   - Рабочая папка: `C:\RomanAlexandrov\signaling-server`

**Вариант B — PM2**

```powershell
npm install -g pm2
cd C:\RomanAlexandrov\signaling-server
pm2 start server.js --name signaling
pm2 save
pm2 startup
```

## 7. HTTPS и wss

Сейчас приложение использует **`ws://176.99.158.181:9090`** (без шифрования).

Чтобы использовать **wss://** (без блокировки на некоторых сетях), нужен reverse-proxy (IIS, nginx, Caddy) с SSL-сертификатом и проксированием WebSocket на `localhost:9090`. Это отдельная настройка; для домашнего IP обычно достаточно ws + разрешение cleartext в приложении для этого хоста.

## 8. Проверка с телефона

В приложении должен быть указан URL (см. `SignalingConfig`):

```text
ws://176.99.158.181:9090
```

Оба устройства (ведущий и ведомый) должны выходить в интернет; сигналинг соединяет их через ваш сервер.
